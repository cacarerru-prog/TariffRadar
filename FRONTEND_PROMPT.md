# 🚀 ПРОМТ ДЛЯ СОЗДАНИЯ ПОЛНОГО REACT ПРИЛОЖЕНИЯ TariffRadar

## Контекст
Дипломный проект — система мониторинга и сравнения тарифов логистических компаний белорусского рынка. 
- **Бэк:** Go, PostgreSQL, Redis (API готов на localhost:8080)
- **Фронт:** React + Tailwind CSS (нужно создать)
- **Цель:** Готовый продукт, 1000+ реальных данных, все работает

---

## 📋 ТРЕБОВАНИЯ К ПРИЛОЖЕНИЮ

### 1. АРХИТЕКТУРА
- **Framework:** React 18 + Hooks (useState, useContext, useEffect)
- **Стили:** Tailwind CSS (встроить через CDN или className)
- **Состояние:** Context API (для авторизации, профиля, выбранных маршрутов)
- **HTTP:** fetch() для API-запросов к `http://localhost:8080/api/v1/*`
- **Хранилище:** localStorage для JWT токена, профиля, избранных маршрутов, фильтров
- **Язык:** Русский
- **Валюта:** EUR (основная), USD, BYN, RUB

### 2. ЭКРАНЫ И ФУНКЦИОНАЛЬНОСТЬ

#### **А) АУТЕНТИФИКАЦИЯ** (`/auth`)
- Страница входа: email + пароль
- Страница регистрации: email + пароль + имя + компания
- После успеха → сохранить JWT в localStorage
- Редирект на Dashboard
- **Кнопка Выход** → стереть токен, вернуться на login

**Компоненты:**
- `LoginPage` — форма входа
- `RegisterPage` — форма регистрации
- `ProtectedRoute` — обёртка для защиты экранов

#### **Б) DASHBOARD** (главная, после логина)
**KPI-карточки:**
- Средняя ставка (avg) — большая цифра
- Минимум (min)

- Количество сделок (deals_count)

**Фильтры (выше карточек):**
- **От:** автодополнение городов (список: Минск, Москва, Казань, Санкт-Петербург...)
- **До:** автодополнение городов
- **Тип:** select FTL | LTL
- **Период:** buttons "7Д" | "30Д" | "90Д" | "1Г"
- **Кнопка:** "Обновить"

**График (ниже карточек):**
- Line chart (recharts или Chart.js)
- X-ось: даты/дни (зависит от периода)
- Y-ось: цены
- Легенда с валютой

**Боковое меню или навигация:**
- Dashboard (активный)
- Search
- Insights
- Deal History
- Add Data
- My Routes
- Benchmark
- Settings

**Сохранение состояния:**
- При выборе фильтров → localStorage
- При refresh → восстановить фильтры и данные

**Компоненты:**
- `Dashboard` — главный контейнер
- `KPICard` — одна карточка
- `FilterBar` — фильтры (города, период, тип)
- `LineChart` — график
- `Sidebar` — навигация

**Компоненты:**

#### **Г) INSIGHTS** (`/insights`)
**3 блока (табы или отдельные секции):**

**1. Тренды (Растущие/Падающие маршруты)**
- Query: period (select 7Д, 30Д, 90Д, 1Г), limit (5, 10, 20)
- Результат: два списка (rising, falling)
- Каждый маршрут: название + change_pct (красный/зелёный)

**2. Сезонность (по месяцам)**
- Фильтры: от, до, тип
- Результат: Area chart
- X-ось: месяцы (Янв, Фев, Мар...)
- Y-ось: цены
- Min/Max информация

**3. По типам груза**
- Фильтры: от, до, тип, период
- Результат: горизонтальные бары (bar chart)
- Каждый груз: FMCG, Металл, Одежда... с ценой

**Компоненты:**
- `InsightsPage`
- `TrendsTab` — растущие/падающие
- `SeasonalityChart` — график по месяцам
- `CargoPricesChart` — бары по грузам

#### **Е) ДОБАВИТЬ ДАННЫЕ** (`/add-data`)
**Форма:**
- От (select из списка городов)
- До (select из списка городов)
- Тип (FTL | LTL)
- Дата (date input, не старше 1 года, не в будущем)
- Тип груза (select: FMCG, Металл, Одежда, Продукты, Химия, Стройматериалы, Мебель)

- Успех → тост уведомление "✓ Сделка добавлена"
- Очистить форму

**Компоненты:**
- `AddDataPage`

### 4. ИНТЕГРАЦИЯ С API

**API-функции (fetch wrapper):**
```javascript
// auth
POST /api/v1/auth/register
POST /api/v1/auth/login
GET /api/v1/auth/me (protected)

// market
GET /api/v1/market/stats?from=...&to=...&type=...&period=...

// deals
GET /api/v1/deals?from=...&to=...&period=...&page=...&per_page=...
POST /api/v1/deals (protected, требует JWT)
GET /api/v1/export/deals?format=csv&period=...
GET /api/v1/export/deals?format=xlsx&period=...

// insights
GET /api/v1/insights/trends?period=...&limit=...
GET /api/v1/insights/seasonality?from=...&to=...&type=...
GET /api/v1/insights/by-cargo?from=...&to=...&type=...&period=...

// benchmark
POST /api/v1/benchmark

// my routes
GET /api/v1/my/routes (protected)
POST /api/v1/my/routes (protected)
DELETE /api/v1/my/routes/:id (protected)

// profile
GET /api/v1/me (protected)
PATCH /api/v1/me (protected)
PATCH /api/v1/me/password (protected)
PATCH /api/v1/me/notifications (protected)
```

**Обработка ошибок:**
- 401 → очистить токен, редирект на login
- 400 → показать ошибку валидации
- 500 → показать generic error toast

---

### 5. ЛОКАЛЬНОЕ ХРАНИЛИЩЕ (localStorage)

Сохранять:
- `token` — JWT токен
- `user` — профиль пользователя (email, name, role)
- `filters` — последние выбранные фильтры (от, до, период, тип)
- `myRoutes` — ID избранных маршрутов (для быстрой загрузки)

При загрузке приложения:
- Если нет токена → показать Login
- Если токен есть → GET /me (валидировать), если ошибка 401 → logout
