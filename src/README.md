# TariffRadar — серверная часть

Серверная часть системы мониторинга и сравнения тарифов логистических компаний. Дипломный проект Качевского А.А., БГУИР, гр. 263102.

## Архитектура

Слоистая архитектура (Clean Architecture):

```
┌─────────────────────────────────────────┐
│  HTTP Layer (handlers + middleware)     │  ← REST API, JWT, CORS
├─────────────────────────────────────────┤
│  Service Layer (business logic)         │  ← Расчёт benchmark, агрегация
├─────────────────────────────────────────┤
│  Repository Layer (data access)         │  ← SQL-запросы, кэширование
├─────────────────────────────────────────┤
│  PostgreSQL 15  +  Redis 7              │
└─────────────────────────────────────────┘
```

## Структура папок

```
src/
├── cmd/server/main.go         — точка входа (запуск HTTP-сервера)
├── internal/
│   ├── config/                — загрузка конфигурации из ENV
│   ├── handlers/              — HTTP-обработчики (по экранам фронта)
│   ├── middleware/            — JWT, CORS, логирование, rate-limit
│   ├── models/                — структуры данных (Deal, Route, User…)
│   ├── repository/            — работа с PostgreSQL и Redis
│   └── service/               — бизнес-логика (агрегации, benchmark)
├── migrations/                — SQL-миграции (goose)
├── docs/                      — спецификация API и схемы БД
├── pkg/                       — переиспользуемые пакеты
├── scripts/                   — скрипты для разработки и сидинга БД
├── docker-compose.yml         — Postgres + Redis для локальной разработки
├── Dockerfile                 — сборка приложения
├── go.mod                     — Go-модули
├── Makefile                   — команды dev-цикла
└── .env.example               — пример переменных окружения
```

## Стек

- **Go 1.21**, `net/http` (stdlib), `chi` для маршрутизации
- **PostgreSQL 15** — основное хранилище
- **Redis 7** — кэш агрегатов
- **goose** — миграции БД
- **swaggo/swag** — OpenAPI документация
- **golang-jwt/jwt** — JWT-аутентификация
- **Docker / Docker Compose** — контейнеризация

## Запуск (после реализации этапа 0–1)

```bash
# 1. Поднять Postgres + Redis
docker-compose up -d

# 2. Применить миграции
make migrate-up

# 3. Запустить сервер
make run

# 4. Проверить
curl http://localhost:8080/health
```
