
## ❌ НЕ ВЫКЛАДЫВАЕМ:

### 1️⃣ **Папка node_modules/** 
❌ Слишком большая (100+ МБ)  
✅ Воссоздаётся через `npm install`  
**Исключено в .gitignore**

### 2️⃣ **Файл .env**
❌ Содержит секреты (DB_PASSWORD, JWT_SECRET, STRIPE_KEY)  
✅ Каждый разработчик создаёт свой  
**Исключено в .gitignore**

### 3️⃣ **Папка diploma/** — дипломная работа
❌ .docx файлы — не text-based, большие, часто меняются  
❌ Это личный документ студента, не часть кода  
✅ Можно хранить отдельно (OneDrive, Google Drive) или в отдельном репозитории  
**ИСКЛЮЧАЕМ из .gitignore**

### 4️⃣ **Папка практика/**
❌ Личные файлы, не часть проекта  
**ИСКЛЮЧАЕМ из .gitignore**

### 5️⃣ **Папка пример-диплома/**
❌ Примеры для справки, не часть кода  
**ИСКЛЮЧАЕМ из .gitignore**

### 6️⃣ **Build артефакты**
```
❌ bin/                    (Go бинарники)
❌ dist/                   (Собранный frontend)
❌ build/                  (Build папки)
❌ *.exe, *.dll            (Windows бинарники)
```
**Исключено в .gitignore**

### 7️⃣ **Логи и временные файлы**
```
❌ *.log                   (логи приложения)
❌ tmp/, temp/             (временные файлы)
❌ .DS_Store               (macOS)
❌ Thumbs.db               (Windows)
```
**Исключено в .gitignore**

### 8️⃣ **IDE файлы** (опционально)
```
❌ .idea/                  (IntelliJ IDEA)
❌ .vscode/                (VS Code)
```
**Исключено в .gitignore** (или создать .vscode/settings.json)

## ✨ Финальный чек-лист перед push

```bash
# 1. Проверить размер
du -sh .

# 2. Посмотреть, что будет загружено
git status

# 3. Проверить .gitignore
git check-ignore -v *

# 4. Если всё ОК — push
git push origin main
```
